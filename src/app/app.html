Quick Start

Get a grid running in under 2 minutes.

Install
npm install ag-grid-angular ag-grid-community
Component
import { Component, signal } from '@angular/core';
import { AgGridAngular } from 'ag-grid-angular';
import { AllCommunityModule, ModuleRegistry } from 'ag-grid-community';

ModuleRegistry.registerModules([AllCommunityModule]);

@Component({
  selector: 'app-my-grid',
  standalone: true,
  imports: [AgGridAngular],
  template: `
    <ag-grid-angular
      style="width: 100%; height: 400px;"
      [rowData]="rowData()"
      [columnDefs]="colDefs()"
    />
  `,
})
export class MyGridComponent {
  rowData = signal([
    { make: 'Tesla', model: 'Model Y', price: 64950 },
    { make: 'Ford',  model: 'F-Series', price: 33850 },
  ]);

  colDefs = signal([
    { field: 'make' },
    { field: 'model' },
    { field: 'price' },
  ]);
}

Best Practices

Patterns that keep grids fast, maintainable, and bug-free.

⚡

Use Signals

Prefer signal() for rowData and colDefs over plain class properties — grids update reactively with no manual change detection.

📦

Lazy Load Components

Load concept components via loadComponent in app.routes.ts to keep the initial bundle small and only pay for what is visited.

🔧

Register Modules Once

Call ModuleRegistry.registerModules() once per component file, at the top level — not inside a lifecycle hook or constructor.

🔑

Define Row IDs

Set getRowId for large or frequently-updated datasets. AG Grid uses it to efficiently apply delta row updates without re-rendering the full grid.

🎨

Theme via CSS Class

Apply ag-theme-alpine or ag-theme-quartz via a CSS class on the component element — never inline. This keeps theming overridable in global styles.

📐

Fixed Grid Height

Always give the grid a fixed px or % height. Using height: auto causes the grid to collapse — it needs a known height to render its virtual rows.
